## Problem statement

Given a permutation group `G ≤ Sym(n)` by a list of generators, compute:

1. A **base** `B = (b_1, …, b_k)`: a sequence of points whose pointwise
   stabiliser in `G` is trivial.
2. A **strong generating set** with respect to `B`.
3. The order `|G|` and the basic transversal sizes `|U_i|`. By
   orbit-stabiliser, `|G| = ∏ |U_i|`.
4. Membership decisions for a list of query permutations.

The construction algorithm is **Schreier-Sims with Sims' filter** — the
incremental version that sifts each Schreier generator against the
partial BSGS and inserts non-trivial residues at the appropriate level.

## I/O contract (JSON)

### Input (one JSON object on stdin)

```jsonc
{
  "degree":              <int, ≥ 1>,
  "generators":          [[<image array of length degree>], …],
  "membership_queries":  [[<image array>], …]   // possibly empty
}
```

A permutation on `n` points is its **0-indexed image array**:
`perm[i]` is the image of `i`. The identity is `[0, 1, …, n−1]`.

### Output (one JSON object on stdout)

```jsonc
{
  "base":                [<int>, …],
  "strong_generators":   [[<image array>], …],
  "transversal_sizes":   [<int>, …],     // |U_1|, |U_2|, …, length = len(base)
  "order":               "<decimal string>",
  "membership_results":  [<bool>, …]      // same length as membership_queries
}
```

`order` is a decimal string; for `M_24` and the larger sporadic groups it
exceeds `2³²`. Strong generators are returned as image arrays.

## Canonical phrasing (informational)

These short excerpts ground definitions and conventions. They are
**informational, not restrictive**.

> 1. *Stabiliser chain and transversals (Sims's notation: G⁽ⁱ⁾ is the
>    stabiliser of {1, …, i}, Uᵢ is a transversal of G⁽ⁱ⁾ in G⁽ⁱ⁻¹⁾):*
>    "Let Uᵢ be a set of right coset representatives for G⁽ⁱ⁾ in G⁽ⁱ⁻¹⁾,
>    1 ≤ i ≤ n−1."
>    — `Sims_PermutationGroups_1970.pdf:p9`
> 2. *Orbit-stabiliser factorisation of |G|:*
>    "nᵢ = |Uᵢ| = |G⁽ⁱ⁻¹⁾ : G⁽ⁱ⁾|. nᵢ is the length of the orbit Δ of
>    G⁽ⁱ⁻¹⁾ containing i and for each j ∈ Δ there is a unique element in
>    Uᵢ taking i to j. Also |G| = ∏ᵢ₌₁ⁿ⁻¹ nᵢ."
>    — `Sims_PermutationGroups_1970.pdf:p9`
> 3. *Schreier generators for the next stabiliser (σ(g) is the Uᵢ
>    representative of the coset G⁽ⁱ⁾g):*
>    "G⁽¹⁾ is generated by C₁ = {ucx σ(ucx)⁻¹ | u ∈ U₁, x ∈ C}."
>    — `Sims_PermutationGroups_1970.pdf:p9`
> 4. *The "Sims filter" idea — keep the generating set small by
>    sifting each new candidate against the existing transversal:*
>    "whenever one of the elements ucx σ(ucx)⁻¹ is computed, the process
>    described above should be used to determine if it can be expressed
>    in terms of the coset representatives already constructed and is
>    therefore redundant."
>    — `Sims_PermutationGroups_1970.pdf:p9`

# Problem 13 — Oracle strategy

The golden master is generated from a **two-oracle consensus** plus
**bug-immune closed-form anchors**. Empirically validated by the
research probe: Wolfram and mpmath agree bytewise to 200 decimal digits
on a generic random parameter set, to 50 decimal digits on coalescent
parameters, and to 50 decimal digits at the four canonical unit-circle
points.

## The two oracles

### Wolfram (`wolframscript`)

Locally available at `/usr/bin/wolframscript` (Wolfram engine 1.13+).
Invoke as

```
wolframscript -code 'N[MeijerG[{a-list, b-list}, {c-list, d-list}, z], digits]'
```

where the four sub-lists encode `(a₁..aₙ, a_{n+1}..aₚ)` and
`(b₁..b_m, b_{m+1}..b_q)` per Mathematica's convention.

**Force numerical-first** by using `N[..., digits + 10]` rather than
`N[MeijerG[...]]`. The reason: `MeijerG[...]` without `N` may invoke
`FunctionExpand` / `FullSimplify` and reduce symbolically before
numerical evaluation; the symbolic-then-numerical path can take
different code than direct numerical Mellin–Barnes contour quadrature
and may produce subtly different rounding behaviour at the boundary
of convergence regions.

### mpmath

Locally available at `python3 -m mpmath` (mpmath 1.4+). Invoke as

```
python3 -c "from mpmath import meijerg, mp; mp.dps=110; print(meijerg([[a₁,...,aₙ],[a_{n+1},...,aₚ]], [[b₁,...,b_m],[b_{m+1},...,b_q]], z))"
```

mpmath's algorithm:

1. Apply Slater's residue theorem analytically once.
2. Build a list of `m` (Series 1) or `n` (Series 2) `pFq` terms with
   prefactors `z^(b_k/r) · ∏Γ(...)/Γ(...)`.
3. Auto-select Series 1 vs Series 2 by `(p, q, m, n, |z|)`:
   - `series=1` if `p < q`;
   - `series=2` if `p > q`;
   - `series=2` if `p == q ∧ m + n == p ∧ |z| > 1`;
   - else `series=1`.
4. Evaluate each `pFq` via `hypercomb` with cancellation monitoring
   and parameter perturbation (`hmag`).

The mpmath value is bit-identical given fixed `dps`, fixed parameters,
and fixed `series` choice.

### Why SymPy is *not* an independent oracle

SymPy's `meijerg._eval_evalf` dispatches to `mpmath.meijerg`. SymPy's
numerical value is mpmath's numerical value. SymPy's contribution is
the Roach-1996/97 symbolic-side `hyperexpand.py` + `meijerint.py` —
that is reference for the algorithm but not an independent numerical
witness.

### Why Arb does not help

Arb (`arb_hypgeom`) does not implement Meijer G natively. Verified by
direct documentation fetch in the research probe. Only `acb_hypgeom_pfq`
is available; a Meijer G via Slater would need to be assembled by the
caller.

## Tier-0 closed-form anchors (bug-immune)

The strongest possible test cases. Both oracle and candidate must
match a *symbolic* closed form. The ground-truth value is computed
from the right-hand side of a known identity, **not** from MeijerG.

| `(m,n,p,q)` | a-list | b-list | Closed form | Source |
|---|---|---|---|---|
| `(1,0,0,1)` | — | `{0}` | `e^{-z}` | DLMF 16.18.1; Bateman §5.6 (1) |
| `(1,0,0,1)` | — | `{a}` | `z^{a/2}·e^{-z}` (Marichev / PBM) | PBM 8.4.3.1 |
| `(1,0,0,2)` | — | `{0, ½}` | `(1/√π)·cos(2√z)` | Bateman §5.6 |
| `(1,0,0,2)` | — | `{½, 0}` | `(1/√π)·sin(2√z)` | Bateman §5.6 |
| `(1,0,0,2)` | — | `{ν/2, −ν/2}` | `J_ν(2√z)` | Bateman §5.6 (16) |
| `(1,0,0,2)` (z<0 form) | — | `{ν/2, −ν/2}` | `I_ν(2√(−z))` | Bateman §5.6 (16') |
| `(2,0,0,2)` | — | `{ν/2, −ν/2}` | `2·K_ν(2√z)` | Bateman §5.6 (20) |
| `(2,0,1,2)` | `{1}` | `{0, ½}` | `(1/√π)·erfc(√z)` | DLMF 16.18 |
| `(1,2,2,2)` | `{1, 1}` | `{1, 0}` | `log(1 + z)` | Bateman §5.6 |
| `(1,2,2,2)` | `{½, 1}` | `{½, 0}` | `arctan(√z)` | Bateman §5.6 |
| `(2,0,1,2)` | `{1}` | `{0, a}` | `Γ(a, z)` (incomplete Γ) | DLMF 16.18, §8.19 |
| `(1,1,1,1)` | `{1−a}` | `{0}` | `(1−z)^{−a} · 1/Γ(a)` | Bateman §5.6 (4) |
| `(1,p,p,q+1)` | `{1−a₁, …, 1−aₚ}` | `{0, 1−b₁, …, 1−b_q}` | `(∏Γ(b_k)/∏Γ(a_j)) · pFq(a₁..aₚ; b₁..b_q; −z)` | DLMF 16.18.1 |

For each anchor, the right-hand side is evaluated in mpmath at 200
decimal digits (e.g. `from mpmath import e, exp, mp; mp.dps=200;
exp(-mpf("2.5"))` for the `e^{-z}` anchor). The value is stored
verbatim in the golden as a decimal string. **No MeijerG call is
involved on the oracle side**; bug in either oracle's MeijerG code
path is invisible to this tier.

The full Tier-0 list (~35 anchors covering elementary, Bessel,
Whittaker, error, exponential-integral, Fresnel, Legendre, Laguerre,
Hermite, Chebyshev, Polylog, hypergeometric pFq) is generated by
`golden/generate.py` (to be written) from a curated YAML of
`(parameter-list, identity-name, rhs-expression)` tuples.

## Tier-1 — Wolfram + mpmath consensus

For every test case in Tiers A through G:

1. Evaluate via `wolframscript -code 'N[MeijerG[...], 110]'` and parse
   the output as an mpc.
2. Evaluate via `python3 -c "from mpmath import meijerg, mp; mp.dps=110;
   print(meijerg([[an], [ap]], [[bm], [bq]], z))"`.
3. Compute relative difference between the two oracle values.
4. **Accept** the test case iff the two agree to `≥ 80 significant
   figures` of the smaller magnitude. Margin against Wolfram's
   variable trailing-digit guard (~10 digits) and mpmath's `~10-bit
   hextra` (~3 decimal digits) plus safety.
5. Store the **average** of the two values as the consensus value
   (or, for cases where they agree to all 110 digits, either value;
   they are byte-identical there).
6. Disagreement (`< 80 sig figs`) **quarantines** the test case (see
   below); do not silently pick a winner.

## Quarantine protocol

The reference implementations probe identified the failure mode: at
`|z| = 1` with `p == q ∧ m + n == p`, Slater's two natural series
both have radius of convergence 1; on the boundary, both diverge
term-wise. mpmath emits `NoConvergence`; Wolfram emits a `MeijerG::hdiv`
warning ("does not exist; arguments not consistent") and returns a
value via analytic continuation that may not match mpmath's
`series=2` choice.

**Quarantine band:** `|z| ∈ [0.99, 1.01] ∧ p == q ∧ m + n == p`.

For test cases falling in the quarantine band:

1. Do **not** include them in the standard golden.
2. If a coalescence-tier case is desired in the band, supply a **third
   witness** by direct Mellin–Barnes contour integration via
   `mpmath.quad` over a hand-deformed steepest-descent contour (or by
   hand-derived Slater-formula reduction with explicit `Γ`/`pFq`
   product evaluation).
3. Document the witness derivation in `golden/quarantine-cases.md`.

## Branch-cut convention

The integral definition (DLMF 16.17.1) contains `z^s = exp(s · log z)`
with the principal branch `log z = log|z| + i·arg z`, `arg z ∈ (−π, π]`.
This convention is **not** documented in DLMF §16.21. The golden master
pins:

- For `z` with `Re(z) ≤ 0 ∧ Im(z) = 0` (the negative real axis), evaluate
  at `z + i·ε` with `ε = 10^{-100}` (above the cut). The candidate is
  expected to use the same convention.
- For complex `z` not on the cut, evaluate at the exact `z`.

## Speed / cost note

Each `wolframscript` invocation has ~0.3-0.5s startup. mpmath at 110 dps
takes ~30-100ms per generic case, ~200-1000ms for parameter-coalescent
cases (perturbation retries). Full golden generation across ~250 test
cases × two oracles ≈ 30-60 minutes of CPU. Acceptable for a one-shot
generation; not acceptable for re-running per trial. Generation script
caches per-input-hash in `golden/cache/`.

## Implementation

`golden/generate.py` (to be written when the substrate ships) is the
single source of truth. It:

1. Reads `golden/test-cases.yaml` (the curated test set).
2. For each Tier-0 case: evaluates the RHS in mpmath at 200 dps; stores.
3. For each Tier A–G case: invokes both oracles at 110 dps; computes
   agreement; stores or quarantines.
4. For each Tier H case (LCG-driven): invokes both oracles at 60 dps;
   computes agreement at relaxed tolerance; stores.
5. Writes `golden/inputs.json` and `golden/expected.json` per the
   verifier protocol.

The script lives in `infra/` of `ts-bench-infra/` (it is benchmark
infrastructure, not problem-13-specific) eventually; for now staged
in `problems/13-meijer-g/golden/`.
